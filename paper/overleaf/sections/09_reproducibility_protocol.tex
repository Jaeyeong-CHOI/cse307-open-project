    Key finding: reasoning--generation dissociation confirmed across all 4 model families (dense + reasoning). CoT PPR reduction is largest in nano (1.00$\to$0.20) but pass rate remains 0/20 for 3 of 4 models.

  \item \textbf{E23: L4 hard-task battery, $n=10$ per cell, 4 models, 3 tasks (2026-03-25).}
    Hypothesis: harder algorithmic tasks amplify pattern blindness.
    Tasks: H1 (merge\_sort, descending), H2 (binary\_search, inverted return), H3 (BFS, reversed traversal).
    Models: \texttt{gpt-4o}, \texttt{gpt-5.4}, \texttt{o3}, \texttt{o4-mini} (120 total runs).
    Result: hypothesis REJECTED. \texttt{gpt-5.4}, \texttt{o3}, \texttt{o4-mini}: 30/30 (100\%); \texttt{gpt-4o}: 24/30 (80\%).
    Key finding: hard tasks provide \emph{explicit I/O specifications} allowing direct behavioral compliance; simple L4 tasks embed an \emph{implicit semantic rule} that models cannot extract. Pattern blindness is specific to implicit rule extraction, not task complexity. Prior entrenchment depth (not algorithmic difficulty) is the key predictor of resistance.
    Files: \texttt{l4-hard-tasks-*.2026-03-25.json}, \texttt{hard-vs-simple-summary-2026-03-25.md}.

  \item \textbf{E21: Failure taxonomy, 419 failed L4 runs (2026-03-25).}
    Classified all failed L4 ablation + multi-task T1 runs across ablation (350) and multi-task (69 failed T1 runs).
    Type-I (exact Python prior): 83.1\%; Type-II (partial attempt): 12.6\%; Type-III (operational substitution): 4.3\%.
    Type-I dominates in \texttt{gpt-4.1}/\texttt{gpt-4.1-nano} (100\%); Type-II concentrates in \texttt{gpt-4.1-mini} (44\%) and \texttt{gpt-4o-mini} (32\%).
    File: \texttt{failure-taxonomy-2026-03-25.json}
    Script: \texttt{scripts/categorize\_failures.py}

  \item \textbf{E25: L3 T2 diagnostic for \texttt{gpt-4.1}, $n=10$ (2026-03-25).}
    Motivation: \texttt{gpt-4.1} achieves only 1/50 on L4 T2 (\texttt{is\_sorted}), despite L3 T2 being solvable for other models. E25 tests whether this failure is (a) L4-specific pattern extraction failure, or (b) general inability to invert \texttt{is\_sorted} semantics.
    Condition: explicit L3 rule (``if-block runs when condition is FALSE''), task \texttt{is\_sorted}, $n=10$.
    Result: \texttt{gpt-4.1} passes 10/10 (PPR=0.0). Model correctly applies inverted conditional logic under explicit rule.
    Interpretation: supports RLHF over-alignment hypothesis---\texttt{gpt-4.1}'s T2 failure is L4-specific (pattern extraction from examples), not a general \texttt{is\_sorted} inversion difficulty.
    File: \texttt{e25-l3-t2-gpt41-n10-2026-03-25.json}
    Script: \texttt{scripts/run\_e25\_l3\_t2\_gpt41.py}

  \item \textbf{E26: L3 Fibonacci diagnostic for \texttt{Qwen3-32B}, $n=5$ (2026-03-25).}
    Motivation: \texttt{Qwen3-32B} fails L4 Fibonacci with PPR=1.00 across all variants. E26 tests whether this reflects (a) L4-specific pattern blindness (same as OpenAI models) or (b) general semantic meta-rule failure (fails explicit L3 rule too).
    Condition: explicit L3 rule (``if-block runs when condition is FALSE''), Fibonacci task, $n=5$, via Groq API.
    Result: 0/5 pass, PPR=0.80. Model fails even when the inversion rule is stated explicitly.
    Interpretation: \texttt{Qwen3-32B} exhibits \emph{general semantic meta-rule failure}---not L4-specific pattern blindness. Distinct from all tested OpenAI models (which pass L3 reliably). Treat as preliminary ($n=5$); full L3/L4 ablation deferred.
    File: \texttt{e26-qwen3-32b-l3-fibonacci.n5.2026-03-25.json}
    Script: \texttt{scripts/run\_e26\_qwen\_l3.py}

  \item \textbf{E24: Frontier model L4 Fibonacci ablation, $n=10$, \texttt{gpt-5.4} and \texttt{o3} (2026-03-25).}
    Motivation: E23 only tested \texttt{gpt-5.4}/\texttt{o3} on hard tasks (I/O-transparent, shallow priors). C2 critique required testing same models on strict L4 Fibonacci to confirm core claim.
    Condition: Variant~A (2ex-baseline), strict example-only, identical to Table~\ref{tab:l4_ablation}.
    \texttt{gpt-5.4}: 0/10 pass, PPR=1.00 (complete prior dominance).
    \texttt{o3}: 0/10 pass, PPR=1.00 (complete prior dominance).
    Key finding: L4 pattern blindness on prior-entrenched Fibonacci holds for both frontier models as of 2026-03-25.
    Files: \texttt{\seqsplit{e24-frontier-l4-fibonacci.gpt-54.n10.2026-03-25.json}},
    \texttt{\seqsplit{e24-frontier-l4-fibonacci.o3.n10.2026-03-25.json}}.
    Script: \texttt{scripts/run\_e24\_frontier\_l4.py}
