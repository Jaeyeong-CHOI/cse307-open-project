\section{Method}
\subsection{Research Questions and Hypotheses}
\textbf{RQ1}: At what language design level does LLM resistance become robust?
\textbf{RQ2}: What failure mechanism is active at each level---prior dominance or pattern blindness?
\textbf{H1}: Token-level alias substitution (L1) creates partial but not complete resistance.
\textbf{H2}: Syntactic inversion (L2) is insufficient---models adopt new syntax when given explicit rules.
\textbf{H3}: Semantic inversion (L3/L4) provides robust resistance, but differs in mechanism by model capacity.

\subsection{Language Level Taxonomy}
We define a four-level design taxonomy:

\begin{table}[!t]
\centering
\caption{Confusion Language Difficulty Levels}
\label{tab:levels}
\begin{tabular}{cll}
\toprule
Level & Design principle & Expected mechanism \\
\midrule
L1 & Token alias substitution & Prior dominance \\
L2 & Syntax structure inversion & Easily adopted \\
L3 & Semantic inversion (explicit) & Capacity-dependent \\
L4 & Semantic inversion (implicit) & Pattern blindness \\
\bottomrule
\end{tabular}
\end{table}

\subsection{L1: Token Alias Substitution}
Python keywords (\texttt{if}, \texttt{for}, \texttt{in}, \texttt{return}, \texttt{def}) are replaced with aliases. Constraints: one-to-one mapping, no alias collision, token-boundary replacement, literal/comment protection.

\subsection{L2: Syntax Structure Inversion}
Function definitions, conditionals, loops, and block delimiters are structurally inverted relative to Python. Example: \texttt{def fib(n):} becomes \texttt{:define fib [n] ->}. Model is given explicit grammar rules.

\subsection{L3: Semantic Inversion (Explicit Rules)}
Python surface syntax is retained but the semantics of if-blocks are inverted: the indented block executes when the condition is \emph{false}. Model is explicitly told this rule.

\subsection{L4: Semantic Inversion (Implicit, Example-Only)}
Same as L3, but no rule is stated. Instead, models receive 2--3 worked examples that demonstrate the inversion. Models must extract the semantic pattern through few-shot reasoning.

\subsection{Evaluation Pipeline}
Each prompt is sent to the model with the appropriate system message. Outputs are judged by the partial judge:
\begin{itemize}
  \item \textbf{KLR} (Keyword Leakage Rate): fraction of alias slots where original keyword was used.
  \item \textbf{ACR} (Alias Compliance Rate): $1 - \text{KLR}$.
  \item \textbf{PSS} (Partial Structural Score): weighted sub-component credit.
  \item \textbf{PIR} (Prior Influence Rate): $\text{KLR} / (\text{KLR} + \text{ACR})$.
  \item \textbf{PPR} (Python Prior Rate, L3/L4): fraction of outputs using Python-style condition (not inverted).
\end{itemize}

\subsection{Threats to Validity}
\textbf{Internal}: judge rules may be stricter than human acceptance; PSS weights are heuristic.
\textbf{Construct}: single benchmark task (fib) limits generalization.
\textbf{External}: results are specific to tested model families and may not generalize.
