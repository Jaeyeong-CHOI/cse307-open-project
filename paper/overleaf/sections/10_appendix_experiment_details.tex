\section{Appendix: Metric Definitions and Prompt Sample}

\subsection{Metric Definitions}
\label{sec:appendix_pss}
\begin{itemize}
  \item \textbf{KLR} (Keyword Leakage Rate, L1): fraction of outputs containing original Python keywords despite alias instructions. Lower = better compliance.
  \item \textbf{SIR} (Syntax Inversion Rate, L2): fraction of outputs retaining Python syntax instead of adopting the specified alternative syntax. 0.0 = full adoption.
  \item \textbf{PPR} (Python Prior Rate, L3/L4): fraction of outputs using standard Python conditional semantics (non-inverted). 1.0 = complete prior dominance; 0.0 = full inversion compliance.
  \item \textbf{PSS} (Partial Structural Score, L1): heuristic composite (alias 0.4, parse 0.25, skeleton 0.25, format 0.1). Exploratory only; not a primary metric.
  \item \textbf{PES} (Prior Entrenchment Score, E31): logprob probability of the canonical Python token at the base-case decision point.
\end{itemize}

\subsection{L4 Variant A Prompt Example (Fibonacci)}
The strict L4 prompt provides two worked examples with inverted conditional semantics and no explicit rule statement:

\begin{lstlisting}
System: You are a helpful coding assistant.

User: Here are two examples in our language:

Example 1:
def is_positive(n):
    if n < 0:
        return True
    return False

# is_positive(5) -> True
# is_positive(-3) -> False

Example 2:
def is_even(n):
    if n % 2 != 0:
        return True
    return False

# is_even(4) -> True
# is_even(7) -> False

Now write a fibonacci function in this
language.
\end{lstlisting}

The judge checks whether the model's output uses inverted comparison operators (\texttt{>}, \texttt{>=}) in the base case rather than the standard Python \texttt{<=}.

\subsection{L1 Pilot Exploration: v1--v120 Researcher-Designed Aliases}
\label{sec:appendix_pilot_l1}

\textbf{Caution: selection bias possible.} The 120 alias prompts (v1--v120) were designed by the researcher without random sampling. The type distribution (21~Type~A, 84~Type~B, 15~Type~C) reflects incidental design choices rather than a controlled experiment. This pilot data is presented for historical context only; the primary L1 finding is the factorial experiment reported in \S\ref{sec:l1_factorial}.

Under ctx-pack delivery (\texttt{gpt-4o}, $n=120$), KLR by alias type in the pilot set is: Type~A = 1.00, Type~B = 0.96, Type~C = 1.00. KLR for collision vs.\ non-collision prompts is 1.00 vs.\ 0.96. These near-ceiling values contrast sharply with the factorial result (Type~C KLR = 0.00), confirming that the pilot's apparent uniformity was an artifact of researcher-selected alias vocabulary.

The 46 collision cases (multiple Python keywords $\to$ same alias, e.g., \texttt{in}/\texttt{while} $\to$ \texttt{ret}) arose incidentally and are not comparable to a controlled collision design. The pilot collision vs.\ non-collision comparison is therefore uninterpretable.
