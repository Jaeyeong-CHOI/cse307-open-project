\section{Experiments}
\subsection{Experimental Overview}
We ran experiments at each language level for available model families. All runs used prompts from \texttt{v1--v120} (L1) or dedicated level-specific banks (L2--L4). The AVRTG key path was used for authenticated API calls.

\subsection{Models Evaluated}
\begin{itemize}
  \item \texttt{gpt-4o}, \texttt{gpt-4o-mini}, \texttt{gpt-4.1-mini}, \texttt{gpt-4.1-nano}, \texttt{gpt-5.4-mini}
\end{itemize}

\subsection{L1 Experiment: Token Alias Substitution}
120 prompts per model, versioned v1--v120 with varied alias mappings. Context-pack ablation: baseline vs. explicit alias contract system prompt.

\subsection{L2 Experiment: Syntax Inversion}
5 prompts using \texttt{:define ... ->} syntax (see Appendix). Model given complete grammar specification.

\subsection{L3 Experiment: Semantic Inversion (Explicit)}
10 prompts. System message explicitly states: ``if-blocks execute when condition is FALSE.'' Evaluated on whether model inverted condition logic.

\subsection{L4 Experiment: Semantic Inversion (Implicit)}
Initial screening: 10 prompts per model, no rule stated; only 2 worked examples with inverted semantics shown. Model must infer pattern.
Ablation (n=50): 5 prompt variants $\times$ 10 runs each, varying the number of worked examples (2 vs.\ 3), presence of inline comments, order of examples, and domain of examples. Evaluated across \texttt{gpt-4o}, \texttt{gpt-4o-mini}, and \texttt{gpt-4.1-mini}. A parallel run on \texttt{gpt-5.4-mini} yielded $n=42$ valid samples (variant E rejected with HTTP 400 due to an unsupported API parameter); results are reported separately in the discussion.
