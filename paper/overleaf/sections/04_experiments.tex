\section{Experiments}
\subsection{Experimental Overview}
We ran experiments at each language level for available model families. All runs used prompts from \texttt{v1--v120} (L1) or dedicated level-specific banks (L2--L4). API authentication was managed via project environment variables.

\subsection{Models Evaluated}
\begin{itemize}
  \item \texttt{gpt-4o}, \texttt{gpt-4o-mini}, \texttt{gpt-4.1}, \texttt{gpt-4.1-mini}, \texttt{gpt-4.1-nano}, \texttt{gpt-5.4-mini}, \texttt{o4-mini}
\end{itemize}

\subsection{L1 Experiment: Token Alias Substitution}
120 prompts per model, versioned v1--v120 with varied alias mappings. Context-pack ablation: baseline vs. explicit alias contract system prompt.

\subsection{L2 Experiment: Syntax Inversion}
5 prompts using \texttt{:define ... ->} syntax (see Appendix). Model given complete grammar specification.

\subsection{L3 Experiment: Semantic Inversion (Explicit)}
10 prompts. System message explicitly states: ``if-blocks execute when condition is FALSE.'' Evaluated on whether model inverted condition logic.

\subsection{L4 Experiment: Semantic Inversion (Implicit)}
Initial screening: 10 prompts per model, no rule stated; only 2 worked examples with inverted semantics shown. Model must infer pattern.
Ablation (n=50): 5 prompt variants $\times$ 10 runs each, varying the number of worked examples (2 vs.\ 3), presence of inline comments, order of examples, and domain of examples. Evaluated across six models at $n=50$: \texttt{gpt-4o}, \texttt{gpt-4o-mini}, \texttt{gpt-4.1}, \texttt{gpt-4.1-mini}, \texttt{gpt-4.1-nano}, and \texttt{gpt-5.4-mini} (300 total runs). One reasoning model (\texttt{o4-mini}) was evaluated at $n=20$ (5 variants $\times$ 4 runs), yielding 320 total L4 ablation runs.

\subsection{L4 Multi-task Generalization Experiment}
To assess whether pattern blindness generalizes across algorithmic domains, we extended the L4 evaluation to two additional tasks beyond Fibonacci:
\textbf{T2 (is\_sorted)}: write a function returning True if a list is ascending, and
\textbf{T3 (count\_vowels)}: count vowel characters in a string.
Both use the identical shared prompt header (two worked examples with inverted semantics, no explicit rule)—the same prompt structure as Ablation Variant A.
We ran $n=10$ per task for \texttt{gpt-4o}, \texttt{gpt-4o-mini}, \texttt{gpt-4.1}, and \texttt{gpt-4.1-nano}; $n=20$ for \texttt{gpt-4.1-mini}; and $n=15$ for \texttt{o4-mini} (225 total multi-task runs across 6 models $\times$ 3 tasks).
