\section{Discussion}
\subsection{Effective LLM-resistant language design}
Our results identify a principled recipe for LLM-resistant programming languages:
\begin{enumerate}
  \item Token-level perturbation alone is insufficient (models partially resist, but at measured rates).
  \item Syntactic restructuring is also insufficient (models adopt explicit structural rules).
  \item Semantic inversion with explicit instruction provides capacity-dependent resistance.
  \item Semantic inversion without explanation (L4) provides consistent resistance via pattern blindness.
  \item \textbf{Task selection is critical}: tasks with deep recursive priors (e.g., Fibonacci) are universally resistant, while tasks with shallower or more varied idioms (e.g., is\_sorted) remain extractable by mid-to-large models. Instructors should prefer tasks with strong, uniform Python prior entrenchment.
  \item \textbf{Prior entrenchment depth governs resistance}: L4 resistance holds only for tasks with deep, uniform Python priors (e.g., Fibonacci recursion template). Complex tasks with varied implementations (merge sort, BFS, binary search) allow capable models (\texttt{gpt-5.4}, \texttt{o3}, \texttt{o4-mini}) to extract inverted semantics from I/O examples---but this is partly confounded by I/O specification transparency (see \S\ref{sec:l4_hard}). Critically, \texttt{gpt-5.4} and \texttt{o3} tested directly on strict L4 Fibonacci (E24) both fail: 0/10 (PPR=1.00)---confirming that even frontier models are defeated by prior-entrenched tasks, not just less capable models. Instructors should prefer tasks with strong, uniform Python prior entrenchment.
\end{enumerate}
The most practical target for educational use is L4: retain Python surface syntax, invert conditional semantics, and provide only worked examples rather than explicit rules---applied to tasks with deep Python priors (e.g., Fibonacci). \textbf{Warning:} harder algorithmic tasks (merge sort, BFS, binary search) do \emph{not} reliably provide L4 resistance because they have both shallower priors \emph{and} more transparent I/O specifications; L4 works for prior-entrenched, pattern-blind tasks.

\subsection{Implications for programming education}

\noindent\textbf{Why human students retain an advantage.}
The practical value of L4 rests on a fundamental asymmetry that does not require experiment to accept.
An instructor who tells students \emph{``in this assignment, if-blocks execute when the condition is false---here are three worked examples''} is providing a rule that a human learner can read, verify against the examples, and apply step-by-step.
The \emph{same} prompt given to an LLM triggers a different mechanism: the model's next-token prediction (NTP) prior---built from billions of standard Python tokens---systematically pulls output back toward \texttt{if n <= 1: return n}, regardless of the stated rule.
This is the core mechanism we measure: not that LLMs are ``less intelligent,'' but that their generation process is governed by distributional priors that cannot be fully overridden by in-context instructions for sufficiently entrenched patterns.
A student \emph{reasons} about a rule; an LLM \emph{samples} from a prior.

Our findings partially address the concern of LLM-solved homework.
L4-style assignments provide meaningful resistance for tasks with deep Python priors (e.g., Fibonacci recursion): all non-reasoning models fail T1 (0/10 to 0/20 pass rates), though via different failure modes: prior dominance (PPR=1.0) for most models, partial template fragmentation (PPR=0.20) for \texttt{gpt-4.1-nano}; o4-mini achieves 8/15 (Wilson 95\% CI: [33\%, 77\%]) only under the partially-annotated multi-task prompt (not under the strict example-only condition); the wide CI reflects the small $n=15$ and should be interpreted as a noisy partial success rather than a definitive estimate.
However, task selection matters: T2 (is\_sorted) is solved by capable models (\texttt{gpt-4o}, \texttt{gpt-4.1-mini} at 100\%), indicating that simpler idioms with shallow priors may remain vulnerable.

Students who understand the L4 rules (from lecture instruction) retain an advantage over LLM-assisted submissions: current models universally fail L4 Fibonacci (T1), while a human who reads the rule and examples can apply it correctly. A formal human study (Exp-5, planned) is needed to quantify the exact human--LLM performance gap; we expect near-ceiling human performance on T1 given the rule sheet, versus 0\% LLM pass rate---a gap large enough to restore discriminative power in practice.

\subsection{Non-monotonic capacity and the gpt-4.1 vs.\ gpt-4.1-mini anomaly}
\label{sec:nonmonotonic}
The most surprising within-family finding is the T2 capacity inversion: \texttt{gpt-4.1-mini} passes 20/20 on \texttt{is\_sorted} (PPR=0.0), while the nominally ``larger'' \texttt{gpt-4.1} achieves only 1/50 (Fisher's exact $p < 0.0001$). We offer two non-exclusive hypotheses (stated as hypotheses, not findings): (a)~\emph{RLHF over-alignment}: the \texttt{gpt-4.1} variant may have been trained with stronger code-generation RLHF reward shaping, causing deeper entrenchment of canonical Python patterns and making prior override more difficult even when pattern cues are present; (b)~\emph{instruction-tuning regime}: the ``mini'' variant may have seen a higher proportion of rule-following fine-tuning data (relative to raw code data), yielding stronger in-context instruction compliance at the cost of raw generation fluency. Both explanations predict that scale alone does not determine pattern extraction ability---fine-tuning decisions interact with model size in complex ways. We flag this as a priority for mechanistic interpretability follow-up.

\subsection{Limits}
While the multi-task experiment (Section~\ref{sec:results}) covers three tasks and seven models (355 total runs including scale-ups), the task set is limited. The task-dependent pass rate for \texttt{gpt-4o} on T2 (is\_sorted: 10/10 pass vs.\ T1 fib: 0/10) confirms that L4 resistance is not unconditional---tasks with shallow or varied Python idioms remain extractable by capable models. The PPR metric for L3/L4 is task-specific and requires per-task oracle semantics. Human baseline experiments (Exp-5) are needed to confirm that L4 does not also confuse human learners.

\smallskip
The \emph{operational substitution} failure mode is confirmed as dominant for T3 at $n=50$: 47/50 \texttt{gpt-4o} runs (94\%; Wilson 95\% CI: [83\%, 99\%]). Our structural judge detects known arithmetic patterns (e.g., \texttt{count -= 1; return -count}), achieving high recall on observed workarounds (47/50 detected). Novel arithmetic transformations not matching these patterns may evade detection; an execution-based judge would provide exhaustive coverage of this failure mode. The scale-up results establish it as real and frequent, not a one-off artifact.

The L4 ablation ($n=50$ per model, 350 total runs) strengthens the pattern blindness finding: zero pass rate holds across all five prompt variants and all seven models including the reasoning model \texttt{o4-mini}. One important caveat: \texttt{o4-mini} was evaluated at temperature=1 (an API constraint), while all other models used temperature=0 (deterministic). Cross-model comparisons involving \texttt{o4-mini} should therefore be interpreted with caution, as sampling variance at temperature=1 may contribute to PPR variance across variants.

\smallskip
The high PPR variance across variants (e.g., \texttt{gpt-4.1-mini} variant B PPR=0.0 vs.\ variant E PPR=1.0) warrants caution---some configurations may trigger different failure sub-modes that are not yet fully characterized. A parallel run of \texttt{gpt-5.4-mini} completed $n=42$ valid evaluations (8 HTTP errors in variant E); all valid runs show PPR=1.0 and pass=0, consistent with full prior dominance across all variants. The overall PPR of 1.00 for \texttt{gpt-5.4-mini} represents the strongest prior dominance among all tested models.

\smallskip
Notably, an earlier preliminary run of \texttt{o4-mini} at $n=20$ yielded 1/20 pass; at $n=50$, this result does not replicate (0/50), indicating that the earlier observation was likely a statistical artifact and not evidence of systematic reasoning-based inversion.

\textbf{Corrected \texttt{o4-mini} result: partial annotation enables reasoning-model extraction.}
An earlier multi-task run of \texttt{o4-mini} used \texttt{max\_completion\_tokens=400}, which truncated reasoning-model responses before the code block was emitted. After correcting to \texttt{max\_completion\_tokens=5000}, \texttt{o4-mini} achieves 8/15 on T1 (partially-annotated prompt). This revision has two implications: (a) the 0/45 result from the strict example-only ablation (Table~\ref{tab:l4_ablation}) is unaffected and confirms pattern blindness under pure implicit delivery; (b) under partial annotation (inline comments that partially articulate the rule), \texttt{o4-mini}'s reasoning capability provides a meaningful advantage that dense models cannot replicate. We report both results transparently. This suggests that the boundary between L3 (explicit rule) and L4 (implicit) is not binary---degrees of partial annotation interact with model reasoning depth.

\subsection{Next steps}
\begin{itemize}
  \item Exp-4 (partial): Multi-task results acquired for T2/T3; broader task coverage (nested control-flow, stateful logic) remains for future work.
  \item Exp-5: Human study comparing student performance vs.\ LLM performance on L4 tasks; critical for validating the educational claim.
  \item Operational substitution: an execution-based judge would improve detection precision beyond the current structural heuristic, particularly for novel arithmetic workarounds beyond the \texttt{count -= 1} pattern.
  \item L4 difficulty scaling: graded version with more/fewer examples, measuring the tipping point at which models begin to extract the inversion pattern.
\end{itemize}
