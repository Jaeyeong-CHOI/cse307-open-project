\section{Discussion}
\subsection{Effective LLM-resistant language design}
Our results identify a principled recipe for LLM-resistant programming languages:
\begin{enumerate}
  \item Token-level perturbation alone is insufficient (models partially resist, but at measured rates).
  \item Syntactic restructuring is also insufficient (models adopt explicit structural rules).
  \item Semantic inversion with explicit instruction provides capacity-dependent resistance.
  \item Semantic inversion without explanation (L4) provides consistent resistance via pattern blindness.
  \item \textbf{Prior entrenchment depth governs resistance}: L4 holds only for tasks with deep, uniform Python priors (e.g., Fibonacci). Complex tasks (merge sort, BFS) allow capable models to extract inverted semantics---partly confounded by I/O transparency (\S\ref{sec:l4_hard}). E24 confirms frontier models (\texttt{gpt-5.4}, \texttt{o3}) also fail on Fibonacci (0/10 each). E31 shows identical PES for fib and merge\_sort yet divergent L4 outcomes, supporting I/O transparency as the critical moderator.
  \item \textbf{Cross-task contamination breaks isolation} (E32, \S\ref{sec:e32_transfer}): L4 resistance is a prompt-level property, not just a task property. Showing inverted code examples from \emph{other} tasks in the same prompt enables models to crack Fibonacci (\texttt{gpt-4.1-mini}: 10/10, up from 0/50 in isolation). Instructors must ensure prompt-level isolation of L4 assignments.
  \item \textbf{Preliminary observation: pattern mimicry may dominate rule following} (E34, \S\ref{sec:e34_density}; preliminary, $n{=}10$, requires replication): showing an inverted code example (0\% verbal annotation) produces higher pass rates than full explicit rules (100\% annotation) across all tested models. This inverts the naive expectation and has direct implications for confusion language design: resistance depends on what the prompt \emph{shows}, not what it \emph{says}.
\end{enumerate}
The most practical target for educational use is L4: retain Python surface syntax, invert conditional semantics, and provide only worked examples rather than explicit rules---applied to tasks with deep Python priors (e.g., Fibonacci). \textbf{Warning:} harder algorithmic tasks (merge sort, BFS, binary search) do \emph{not} reliably provide L4 resistance because they have both shallower priors \emph{and} more transparent I/O specifications; L4 works for prior-entrenched, pattern-blind tasks.

\textbf{Two complementary resistance strategies.} (1)~\emph{L4 prior exploitation}: invert semantics implicitly on deep-prior tasks. (2)~\emph{I/O opacity} (E35): design tasks with opaque mappings where no training distribution provides a shortcut. These target different failure modes and can be combined. The core pattern-blindness claim rests on the strict example-only ablation (9 models, 0/350 pass); multi-task and hard-task extensions provide enriching context but are not directly comparable.

\subsection{Implications for programming education}

The following implications characterize LLM behavior under our experimental conditions. They do not constitute a classroom evaluation; educational outcomes, grading overhead, and adoption feasibility require empirical study in actual instructional settings.

\noindent\textbf{Why human students retain an advantage.}
L4 rests on a fundamental asymmetry: a human who reads ``if-blocks execute when the condition is false'' can apply the rule step-by-step; an LLM's NTP prior---built from billions of Python tokens---systematically pulls output back to \texttt{if n <= 1: return n}. A student \emph{reasons} about a rule; an LLM \emph{samples} from a prior.

L4 assignments provide meaningful resistance for prior-entrenched tasks: all non-reasoning models fail T1 (0/10--0/20); \texttt{o4-mini} achieves 8/15 only under partial annotation (not strict example-only). Task selection matters: T2 is solved by capable models, indicating shallow-prior tasks remain vulnerable. A formal human pilot study would add quantitative precision but is not required to establish the core resistance claim.

\subsection{Non-monotonic capacity and the gpt-4.1 vs.\ gpt-4.1-mini anomaly}
\label{sec:nonmonotonic}
The most surprising within-family finding is the T2 capacity inversion: \texttt{gpt-4.1-mini} passes 20/20 on \texttt{is\_sorted} (PPR=0.0), while the nominally ``larger'' \texttt{gpt-4.1} achieves only 1/50 (Fisher's exact $p < 0.0001$). We offer two non-exclusive hypotheses (stated as hypotheses, not findings): (a)~\emph{RLHF over-alignment}: the \texttt{gpt-4.1} variant may have been trained with stronger code-generation RLHF reward shaping, causing deeper entrenchment of canonical Python patterns and making prior override more difficult even when pattern cues are present; (b)~\emph{instruction-tuning regime}: the ``mini'' variant may have seen a higher proportion of rule-following fine-tuning data (relative to raw code data), yielding stronger in-context instruction compliance at the cost of raw generation fluency.

\textbf{E25 diagnostic (L3 T2 for gpt-4.1):} To discriminate between a general inability to invert \texttt{is\_sorted} semantics versus an L4-specific pattern extraction failure, we ran $n=10$ L3 T2 evaluations for \texttt{gpt-4.1} (explicit rule, same task). Result: \texttt{gpt-4.1} passes 10/10 (PPR=0.0). The model correctly applies the L3 semantic inversion rule for \texttt{is\_sorted} when given an explicit instruction---the exact failure mode it exhibits on L4 T2 (1/50) is absent under L3. This result directly supports hypothesis~(a): the RLHF over-alignment explanation. \texttt{gpt-4.1}'s T2 failure is not due to difficulty understanding inverted \texttt{is\_sorted} semantics; it is specifically a \emph{pattern extraction failure under implicit delivery}---the model cannot extract the inversion rule from worked examples, even though it can follow it from an explicit rule statement. Both explanations predict that scale alone does not determine pattern extraction ability---fine-tuning decisions interact with model size in complex ways. We flag this as a priority for mechanistic interpretability follow-up.

We acknowledge that the partially-annotated multi-task prompt was not pre-specified for the gpt-4.1 series, and interaction effects between prompt design and model variant cannot be excluded. We flag this for follow-up and do not assert causal mechanisms without additional mechanistic evidence.

\subsection{Open-weight model failure modes}
\label{sec:open_weight_discussion}
The open-weight results reveal two distinct failure modes absent from the OpenAI family (see \S\ref{sec:open_weight} for full data and CIs). \texttt{Llama-3.3-70B} exhibits \emph{context-sensitive prior reversion} (introduced in \S\ref{sec:open_weight}): variant-level PPR varies from 0.00 on in-domain examples to 1.00 on domain-shifted examples, suggesting the model's prior engagement is gated by contextual similarity rather than rule-following capacity. The implication for assessment design is direct: any LLM with context-sensitive prior reversion would require \emph{variant diversity} to be reliably fooled---a single example set may inadvertently select a variant that triggers prior reversion recovery. \texttt{Qwen3-32B} exhibits \emph{general semantic meta-rule failure}: the model fails even under L3 (explicit rule), with 0/5 pass and PPR=0.80 (E26 pilot), \emph{confirmed at $n=20$} (E27: 0/20 pass, PPR=0.40, Wilson 95\% CI [0\%, 17\%]). The lower PPR at $n=20$ compared to E26 reflects that many failures are \emph{confused partial instruction-following}---internally inconsistent outputs rather than clean Python prior returns---suggesting a richer failure mode than simple prior dominance. This contrasts sharply with all tested OpenAI models, which pass L3 consistently---including \texttt{gpt-4.1} (10/10, E25)---but fail L4. The Qwen result suggests that L3/L4 failure taxonomy is architecture-dependent: some models exhibit L4-specific pattern blindness (rule-following capability present, pattern extraction absent), while others exhibit L3-level instruction-following failure for semantic inversions (a deeper deficit). Educators using open-weight models should test L3 compliance before assuming that L4 resistance is the binding constraint.

\textbf{Assessment scope for L3-failing models.} For models that fail L3 (e.g., \texttt{Qwen3-32B}), L4 resistance is vacuous from an assessment design perspective: the model is already blocked at the more fundamental level of explicit-rule following, not by the L4 implicit-delivery challenge. L4-style assessment design is most meaningful when applied to models that pass L3 (instruction-following capable) but fail L4 (pattern extraction absent)---i.e., models where the failure is specifically the inability to extract inverted semantics from examples, not a general deficit. Instructors designing L4-resistant assessments should first verify L3 compliance for any LLM they wish to target, to confirm that the assessment challenge is the intended one.

\subsection{Limits}
The task set is limited to three tasks and seven models (345 multi-task runs). L4 resistance is not unconditional: tasks with shallow Python idioms (e.g., \texttt{is\_sorted}) remain extractable by capable models.

\textbf{Temperature caveat.} \texttt{o4-mini} was evaluated at temperature=1 (API constraint); all other models used temperature=0. Three temperature regimes appear: $t{=}0$ (main L4 ablation, multi-task, CoT), $t{=}0.7$ (E32, E34, E35), and $t{=}1$ (\texttt{o4-mini}). Core claims (0/350 fib, 0/30 I/O opacity) rest on $t{=}0$ conditions.

\textbf{Judge validation.} The structural judge achieves 96\% agreement with manual inspection (48/50; \S4). Novel operational substitution workarounds beyond known patterns may evade detection; an execution-based judge would improve coverage.

\textbf{Task-set limitation.} Broader task coverage (nested control-flow, stateful logic) and a human baseline study (Exp-5) remain for future work.

\textbf{Cross-task contamination.} E32 shows that cross-task inverted examples break L4 isolation (\texttt{gpt-4.1-mini}: 10/10 vs.\ 0/50 in isolation). Instructors must ensure prompt-level isolation.

\textbf{Corrected \texttt{o4-mini} result.} An earlier multi-task run used \texttt{max\_completion\_tokens=400}, truncating responses. After correction to 5000, \texttt{o4-mini} achieves 8/15 on T1 under partial annotation---but 0/45 under strict example-only delivery remains, confirming pattern blindness under pure implicit conditions.

\subsection{Next steps}
\begin{itemize}
  \item Exp-4 (partial): Multi-task results acquired for T2/T3; broader task coverage (nested control-flow, stateful logic) remains for future work.
  \item Exp-5: Human study comparing student performance vs.\ LLM performance on L4 tasks; critical for validating the educational claim.
  \item Operational substitution: an execution-based judge would improve detection precision beyond the current structural heuristic, particularly for novel arithmetic workarounds beyond the \texttt{count -= 1} pattern.
  \item Annotation density gradient: E28 established non-monotonic density for \texttt{o4-mini}; E34 (\S\ref{sec:e34_density}) extends to \texttt{gpt-4o} and \texttt{gpt-4.1-mini}, confirming the pattern generalizes across architectures. Follow-up: test \texttt{o3} and other reasoning models; vary prior entrenchment depth to determine whether the saturation threshold shifts with task difficulty.
  \item Prior-depth vs.\ I/O-transparency confound: E31 (\S\ref{sec:e31_pes}) shows both fib and merge\_sort have PES$\geq$0.9996 yet diverge on L4 resistance, supporting I/O transparency as the critical moderator. E33's confounded design has been addressed by E35 (\S\ref{sec:e33_io}), which shows that opaque I/O pairs alone (no algorithm description) produce universal resistance (0/30, 0\%) independent of prior strength. Together, E31+E35 support a two-factor model of resistance: prior entrenchment depth and I/O transparency are both sufficient (but distinct) sources of model failure. E35 establishes I/O opacity as sufficient for resistance on a separate (opaque\_counter) task; whether removing I/O transparency from the hard tasks (H1--H3) would recover resistance remains untested and is the most direct experiment to resolve the hard-task confound.
  \item Cross-task contamination mitigation: E32 (\S\ref{sec:e32_transfer}) shows that cross-task inverted examples break L4 isolation. Future work should test whether prompt-level firewalls (e.g., system-message separation of examples from target task) can restore resistance.
\end{itemize}
