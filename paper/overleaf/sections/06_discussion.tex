\section{Discussion}
The current stage contributes a robust benchmark scaffold rather than definitive model-ranking results.

\subsection{Validated contributions}
We confirmed replay-side reproducibility: fixture summaries and metric snapshots are generated, versioned, and schema-checked. CLI expansion for planner/state outputs reduced parser ambiguity and enabled compact machine-parsable artifacts.

\subsection{Remaining limits}
\begin{itemize}
  \item Live model comparisons are blocked by API authentication issues (401), so cross-model conclusions are preliminary.
  \item Real-world task breadth is still limited due to restricted batch expansion.
\end{itemize}

\subsection{Methodological implication}
For constrained-language evaluation, provenance and schema control are first-order requirements. Without them, failure diagnosis is easily confounded by tooling drift or lineage mismatch, not model behavior.
