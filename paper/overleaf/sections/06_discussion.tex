\section{Discussion}
\subsection{Effective LLM-resistant language design}
Our results identify a principled recipe for LLM-resistant programming languages:
\begin{enumerate}
  \item Token-level perturbation alone is insufficient (models partially resist, but at measured rates).
  \item Syntactic restructuring is also insufficient (models adopt explicit structural rules).
  \item Semantic inversion with explicit instruction provides capacity-dependent resistance.
  \item Semantic inversion without explanation (L4) provides consistent resistance via pattern blindness.
  \item \textbf{Task selection is critical}: tasks with deep recursive priors (e.g., Fibonacci) are universally resistant, while tasks with shallower or more varied idioms (e.g., is\_sorted) remain extractable by mid-to-large models. Instructors should prefer tasks with strong, uniform Python prior entrenchment.
\end{enumerate}
The most practical target for educational use is L4: retain Python surface syntax, invert conditional semantics, and provide only worked examples rather than explicit rules---applied to tasks with deep Python priors.

\subsection{Implications for programming education}
Our findings partially address the concern of LLM-solved homework.
L4-style assignments provide meaningful resistance for tasks with deep Python priors (e.g., Fibonacci recursion): all non-reasoning models fail T1 universally; o4-mini achieves 8/15 only under the partially-annotated multi-task prompt (not under the strict example-only condition).
However, task selection matters: T2 (is\_sorted) is solved by capable models (gpt-4o, gpt-4.1-mini at 100\%), indicating that simpler idioms with shallow priors may remain vulnerable.
The general mechanism for LLM failure on L4 is:
(1) the model defaults to Python prior when encountering familiar syntax, and
(2) even when examples are provided, pattern extraction of semantic inversion is beyond current few-shot capability for sufficiently prior-entrenched patterns.

Students who understand the L4 rules (from lecture instruction) \emph{should} retain an advantage over LLM-assisted submissions in principle, since current models universally fail L4. However, this claim requires empirical validation: a formal human study (Exp-5, planned) is needed to confirm that L4 tasks do not also confuse human learners, and that the human--LLM performance gap is large enough to restore discriminative power in practice.

\subsection{Limits}
While the multi-task experiment (Section~\ref{sec:results}) covers three tasks and seven models (355 total runs including scale-ups), the task set is limited. The task-dependent pass rate for \texttt{gpt-4o} on T2 (is\_sorted: 10/10 pass vs.\ T1 fib: 0/10) confirms that L4 resistance is not unconditional---tasks with shallow or varied Python idioms remain extractable by capable models. The PPR metric for L3/L4 is task-specific and requires per-task oracle semantics. Human baseline experiments (Exp-5) are needed to confirm that L4 does not also confuse human learners. The \emph{operational substitution} failure mode is confirmed as dominant for T3 at $n=50$: 47/50 gpt-4o runs (94\%; Wilson 95\% CI: [83\%, 99\%]). Our structural judge detects known arithmetic patterns (e.g., \texttt{count -= 1; return -count}); novel workarounds not matching these heuristics may evade detection. An execution-based judge would provide exhaustive coverage of this failure mode. However, the scale-up results establish it as real and frequent, not a one-off artifact.

The L4 ablation ($n=50$ per model, 350 total runs) strengthens the pattern blindness finding: zero pass rate holds across all five prompt variants and all seven models including the reasoning model o4-mini. One important caveat: o4-mini was evaluated at temperature=1 (an API constraint), while all other models used temperature=0 (deterministic). Cross-model comparisons involving o4-mini should therefore be interpreted with caution, as sampling variance at temperature=1 may contribute to PPR variance across variants. However, the high PPR variance across variants (e.g., gpt-4.1-mini variant B PPR=0.0 vs.\ variant E PPR=1.0) warrants caution---some configurations may trigger different failure sub-modes that are not yet fully characterized. A parallel run of gpt-5.4-mini completed $n=42$ valid evaluations (8 HTTP errors in variant E); all valid runs show PPR=1.0 and pass=0, consistent with full prior dominance across all variants. The overall PPR of 1.00 for gpt-5.4-mini represents the strongest prior dominance among all tested models. Notably, an earlier preliminary run of o4-mini at $n=20$ yielded 1/20 pass; at $n=50$, this result does not replicate (0/50), indicating that the earlier observation was likely a statistical artifact and not evidence of systematic reasoning-based inversion.

\textbf{Corrected o4-mini result: partial annotation enables reasoning-model extraction.}
An earlier multi-task run of o4-mini used \texttt{max\_completion\_tokens=400}, which truncated reasoning-model responses before the code block was emitted. After correcting to \texttt{max\_completion\_tokens=5000}, o4-mini achieves 8/15 on T1 (partially-annotated prompt). This revision has two implications: (a) the 0/45 result from the strict example-only ablation (Table~\ref{tab:l4_ablation}) is unaffected and confirms pattern blindness under pure implicit delivery; (b) under partial annotation (inline comments that partially articulate the rule), o4-mini's reasoning capability provides a meaningful advantage that dense models cannot replicate. We report both results transparently. This suggests that the boundary between L3 (explicit rule) and L4 (implicit) is not binary---degrees of partial annotation interact with model reasoning depth.

\subsection{Next steps}
\begin{itemize}
  \item Exp-4 (partial): Multi-task results acquired for T2/T3; broader task coverage (nested control-flow, stateful logic) remains for future work.
  \item Exp-5: Human study comparing student performance vs.\ LLM performance on L4 tasks; critical for validating the educational claim.
  \item Operational substitution: an execution-based judge would improve detection precision beyond the current structural heuristic, particularly for novel arithmetic workarounds beyond the \texttt{count -= 1} pattern.
  \item L4 difficulty scaling: graded version with more/fewer examples, measuring the tipping point at which models begin to extract the inversion pattern.
\end{itemize}
