\section{Discussion}
\subsection{Effective LLM-resistant language design}
Our results identify a principled recipe for LLM-resistant programming languages:
\begin{enumerate}
  \item Token-level perturbation alone is insufficient (models partially resist, but at measured rates).
  \item Syntactic restructuring is also insufficient (models adopt explicit structural rules).
  \item Semantic inversion with explicit instruction provides capacity-dependent resistance.
  \item Semantic inversion without explanation (L4) provides consistent resistance via pattern blindness.
\end{enumerate}
The most practical target for educational use is L4: retain Python surface syntax, invert conditional semantics, and provide only worked examples rather than explicit rules.

\subsection{Implications for programming education}
Our findings directly address the concern of LLM-solved homework.
Assignments written in L4-style confusion languages cannot be trivially solved by current LLMs because:
(1) the model defaults to Python prior when encountering familiar syntax, and
(2) even when examples are provided, pattern extraction of semantic inversion is beyond current few-shot capability.

Students who understand the L4 rules (from lecture instruction) retain a decisive advantage over LLM-assisted submissions, restoring the discriminative power of programming assessments.

\subsection{Limits}
While the multi-task experiment (Section~\ref{sec:results}) extends our evaluation beyond Fibonacci, the sample is limited to three tasks and two models ($n=10$ each). The task-dependent pass rate for \texttt{gpt-4o} on T2 (is\_sorted: 10/10 pass vs.\ T1 fib: 0/10) suggests that L4 resistance is not unconditional---tasks whose example patterns closely map to standard Python idioms may be more susceptible to correct inversion. The PPR metric for L3/L4 is task-specific and requires per-task oracle semantics. Human baseline experiments (Exp-5) are needed to confirm that L4 does not also confuse human learners. The newly observed \emph{operational substitution} failure mode (Section~\ref{sec:results}) warrants further characterization: it achieves correct output without adopting L4 semantics, and its detectability may require execution-based rather than structural judgment.

The L4 ablation ($n=50$ per model) strengthens the pattern blindness finding: zero pass rate holds across all five prompt variants and all four models. However, the high PPR variance across variants (e.g., gpt-4.1-mini variant B PPR=0.0 vs.\ variant E PPR=1.0) warrants caution---some configurations may trigger different failure sub-modes that are not yet fully characterized. A parallel run of gpt-5.4-mini completed $n=42$ valid evaluations (8 HTTP errors in variant E); all valid runs show PPR=1.0 and pass=0, consistent with full prior dominance across all variants. The overall PPR of 1.00 for gpt-5.4-mini represents the strongest prior dominance among all tested models.

\subsection{Next steps}
\begin{itemize}
  \item Exp-4 (partial): Multi-task results acquired for T2/T3; broader task coverage (nested control-flow, stateful logic) remains for future work.
  \item Exp-5: Human study comparing student performance vs.\ LLM performance on L4 tasks; critical for validating the educational claim.
  \item Operational substitution characterization: execution-based judge to detect cases where correct output is achieved without L4 rule adoption.
  \item L4 difficulty scaling: graded version with more/fewer examples, measuring the tipping point at which models begin to extract the inversion pattern.
\end{itemize}
