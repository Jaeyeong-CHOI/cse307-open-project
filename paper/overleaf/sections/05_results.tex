\subsection{L1: Partial resistance; scale does not predict compliance}
All models show non-zero KLR (0.26--0.69), confirming prior dominance. Two findings stand out: (1) gpt-5.4-mini (KLR=0.67) underperforms gpt-4o (KLR=0.37), showing model scale does not predict compliance; (2) context-pack delivery halves prior influence (0.42 $\to$ 0.26 for gpt-4o-mini).

\subsection{L2: Syntax inversion not resistant}
All models adopted L2 syntax fully (SIR=0.0) when given explicit rules. Structural novelty alone does not create LLM resistance.

\subsection{L3: Capability threshold at semantic rule-following}
With an explicit semantic inversion rule, gpt-4o-mini fails completely (PPR=1.0): it generates standard Python despite instruction. gpt-4o and gpt-4.1-mini succeed (PPR=0.0). This reveals a model capacity threshold for following semantic meta-rules.

\subsection{L4: Pattern blindness across all tested models}
Without explicit rule, gpt-4o---which succeeds at L3---fails at L4 (PPR=1.0), producing canonical Python and ignoring example-embedded inversion cues. Figure~\ref{fig:l4_failure} shows the contrast.

\textbf{L4 ablation (n=50 per model).}
We scaled the L4 evaluation to $n=50$ runs using five prompt variants across all four models.
Results are shown in Table~\ref{tab:l4_ablation}.
All models achieve 0/50 pass rate regardless of variant, confirming pattern blindness is robust and model-independent.
PPR values (rate of producing Python-prior output) range from 0.38 (gpt-4.1-mini) to 0.84 (gpt-5.4-mini).
Notably, variant E (3ex-diff-domain) induces PPR=1.0 on gpt-4o-mini and gpt-4.1-mini, and near-zero on gpt-5.4-mini (PPR=0.20)---suggesting that when example domain shifts, models may adopt a different surface form rather than inferring inversion, but \emph{still} produce incorrect semantics (pass=0).

\subsection{Two Failure Modes}
\begin{itemize}
  \item \textbf{Prior Dominance} (L1, L3-weak): stated rule is overridden by pretrained token distribution.
  \item \textbf{Pattern Blindness} (L4): model cannot extract semantic rule from examples. Critically, this affects gpt-4o, which follows the same rule when stated explicitly.
\end{itemize}
