\subsection{L1: Partial resistance; scale does not predict compliance}
All models show non-zero KLR (0.21--0.69), confirming prior dominance. Three findings stand out: (1) gpt-5.4-mini (KLR=0.67) underperforms gpt-4o (KLR=0.37), showing model scale does not predict compliance; (2) context-pack delivery consistently reduces KLR---gpt-4.1-mini achieves the lowest KLR (0.21 vs.\ 0.43 baseline, $-51\%$); (3) the reasoning model o4-mini (KLR=0.27) performs comparably to gpt-4o-mini ctx-pack (0.26), suggesting chain-of-thought reasoning provides modest benefit for alias compliance.

\subsection{L2: Syntax inversion not resistant}
All models adopted L2 syntax fully (SIR=0.0) when given explicit rules. Structural novelty alone does not create LLM resistance.

\subsection{L3: Capability threshold at semantic rule-following}
With an explicit semantic inversion rule, gpt-4o-mini fails completely (PPR=1.0): it generates standard Python despite instruction. gpt-4o and gpt-4.1-mini succeed (PPR=0.0). This reveals a model capacity threshold for following semantic meta-rules.

\subsection{L4: Pattern blindness across all tested models}
Without explicit rule, gpt-4o---which succeeds at L3---fails at L4 (PPR=1.0), producing canonical Python and ignoring example-embedded inversion cues. Figure~\ref{fig:l4_failure} shows the contrast.

\textbf{L4 ablation (n=50 per model).}
We scaled the L4 evaluation across seven models: four at $n=50$ and three additional models (gpt-4.1, gpt-4.1-nano, o4-mini) at $n=20$.
Results are shown in Table~\ref{tab:l4_ablation}.
Six of seven models achieve 0\% pass rate; the sole exception is \texttt{o4-mini} (1/20 pass on variant E), suggesting that chain-of-thought reasoning may provide a narrow pathway to overcoming pattern blindness.
PPR values range from 0.38 (gpt-4.1-mini) to 1.0 (gpt-4.1, gpt-4.1-nano, gpt-5.4-mini).
Notably, gpt-4.1 and gpt-4.1-nano show PPR=1.0 across \emph{all} variants, indicating complete prior dominance with no evidence of inversion detection.
The o4-mini result is particularly interesting: as a reasoning model, it partially overcomes pattern blindness where standard models cannot, though only on the most diverse prompt variant (E, different-domain examples).

\subsection{L4 Multi-task Generalization}
To assess whether pattern blindness extends beyond Fibonacci, we evaluated four models on three tasks: T1 (\texttt{fibonacci}), T2 (\texttt{is\_sorted}), and T3 (\texttt{count\_vowels}), using the identical example-only prompt structure. \texttt{gpt-4o} and \texttt{gpt-4o-mini} were evaluated at $n=10$ per task; \texttt{gpt-4.1-mini} at $n=20$ per task. Results are in Table~\ref{tab:l4_multitask}.

Four findings emerge. (1) \textbf{T2 is consistently extractable across capable models}: both gpt-4o (10/10) and gpt-4.1-mini (20/20) pass T2, suggesting \texttt{is\_sorted} maps naturally onto the \texttt{return True/False inside condition} example pattern. (2) \textbf{T1 (Fibonacci) is universally resistant} to L4 extraction: all three models yield PPR=1.0 (or template fragmentation for gpt-4o-mini), confirming deep prior entrenchment for the recursive Fibonacci pattern. (3) \textbf{T3 reveals operational substitution}: gpt-4o uses \texttt{count -= 1; return -count}, achieving correct arithmetic while ignoring the semantic inversion rule---a distinct workaround indicating goal awareness but not rule adoption. gpt-4.1-mini shows partial prior revert (PPR=0.25) yet still fails T3, suggesting this task occupies an intermediate difficulty tier. (4) \textbf{Model capacity determines whether T2 is extractable}: gpt-4o-mini (PPR=1.0 on T2) fails where gpt-4o and gpt-4.1-mini succeed, consistent with the capacity threshold observed in L3. Note: o4-mini evaluation of the multitask suite is ongoing ($n=45$, 0/45 passed so far across T1 and T2); results will be incorporated in the final revision.

\subsection{Three Failure Modes}
\begin{itemize}
  \item \textbf{Prior Dominance} (L1, L3-weak, L4-T2-mini, L4-T1-all): stated or implicit rule is overridden by pretrained token distribution.
  \item \textbf{Pattern Blindness} (L4-T1, L4-T3): model cannot extract the semantic rule from examples alone, even when the same model follows the rule when stated explicitly (L3).
  \item \textbf{Operational Substitution} (L4-T3-gpt-4o): model compensates via arithmetic output transformation rather than adopting the semantic inversion rule, indicating goal awareness without rule internalization.
\end{itemize}
Task difficulty interacts with model capacity to determine which failure mode dominates. T2 (is\_sorted) is extractable by mid-to-large models; T1 (Fibonacci) is resistant for all tested models; T3 (count\_vowels) exposes a third mode unique to models that partially resist prior dominance.
