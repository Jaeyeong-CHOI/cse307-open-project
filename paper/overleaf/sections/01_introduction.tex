\section{Introduction}
Modern code-generation benchmarks~\cite{chen2021codex,austin2021program} show that large language models (LLMs) can solve standard programming tasks with high accuracy.
This creates a critical challenge for programming education: if LLMs solve assignments faster and more accurately than students, how can instructors meaningfully assess student understanding?

This paper studies a complementary but understudied question:
\begin{quote}
\small\textit{Is it possible to design a programming language that is human-interpretable but systematically defeats LLM next-token-prediction priors---making it resistant to AI-based code generation?}
\end{quote}

\noindent\textbf{The core asymmetry.}
The key insight is that humans and LLMs fail in fundamentally different ways when given an unfamiliar language rule.
A human student who reads the rule \emph{``if-blocks execute when the condition is \textbf{false}''} can immediately apply it---the rule is explicit, verifiable, and overrides prior habit.
An LLM given the identical rule in its prompt instead produces standard Python: its next-token prediction (NTP) prior, built from billions of Python training tokens, systematically overrides the stated instruction.
This asymmetry---\emph{humans follow novel rules; LLMs revert to training priors}---motivates our resistance framework as a testable hypothesis rather than a settled premise. If NTP priors dominate generation, then implicit semantic inversions should fail more often than explicit rule statements; our L3/L4 experiments are designed to evaluate exactly this prediction.

We decompose this into three sub-questions:
\begin{enumerate}
  \item Does changing token identifiers (L1: alias substitution) create effective LLM resistance?
  \item Does inverting syntactic structure (L2) create effective resistance?
  \item Does inverting semantic behavior (L3/L4) create deeper resistance?
\end{enumerate}

Our contributions are:
\begin{enumerate}
  \item \textbf{L4 Pattern Blindness: near-universal failure with CoT dissociation.} L4 (implicit semantic inversion) defeats \emph{all} eleven tested models---including frontier models \texttt{gpt-5.4} and \texttt{o3}---on prior-entrenched tasks (0/350 pass on Fibonacci under strict example-only delivery). Chain-of-thought ablation confirms \emph{reasoning--generation dissociation}: models articulate the inversion rule yet still generate standard Python, establishing that the failure is generative, not comprehension-based.
  \item \textbf{An L1--L4 resistance taxonomy with three failure mechanisms and I/O transparency as a second pathway.} We identify \emph{prior dominance}, \emph{pattern blindness}, and \emph{operational substitution}. Token-level (L1) and syntactic (L2) perturbations are insufficient; semantic inversion (L3/L4) is required for robust resistance, modulated by prior entrenchment depth. I/O opacity (E35) provides an independent, complementary resistance source: opaque I/O pairs alone produce complete resistance (0/30) regardless of prior strength.
  \item \textbf{Systematic empirical characterization: 2,000+ evaluations, 11 models, 35 experiments.} Nine OpenAI + two open-weight models, three tasks, and ablations on annotation density, cross-task transfer, and chain-of-thought---providing the first systematic map of where and why NTP-prior-based resistance succeeds or fails.
\end{enumerate}
