\section{Introduction}
Recent studies on AI-assisted coding indicate a persistent tradeoff: immediate coding productivity may rise while durable comprehension and independent debugging skills lag, especially for beginners. This motivates evaluation under constrained syntax settings rather than only canonical benchmarks.

This study asks a strict question: when a model is instructed to produce code in a Python-like language with transformed syntax, does it follow the transformed language rules, or revert to canonical Python conventions?

We define a three-layer benchmark stack:
\begin{enumerate}
  \item \textbf{Specification layer}: one-to-one token aliases and collision constraints.
  \item \textbf{Execution layer}: OCaml-based transform, validate, and roundtrip tooling.
  \item \textbf{Measurement layer}: summary validation, schema checks, and metric snapshots.
\end{enumerate}

Our goal is not absolute model ranking. We isolate where constrained-language compliance fails and how failures relate to execution outcomes.
