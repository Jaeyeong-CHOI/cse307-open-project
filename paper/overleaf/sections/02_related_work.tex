\section{Related Work}
\subsection{Code-Generation Evaluation and LLM Priors}
Early benchmarks for code generation (HumanEval~\cite{chen2021codex}, MBPP~\cite{austin2021program}) established that LLMs trained on large code corpora learn strong token-level priors: Python keywords appear in statistically predictable positions.
Pre-training models such as CodeBERT~\cite{feng2020codebert}, CodeT5~\cite{wang2021codet5}, and UniXcoder~\cite{guo2022unixcoder} reinforce these priors via large-scale NTP objectives.
InCoder~\cite{fried2022incoder} showed that bidirectional context can improve local token prediction but does not eliminate reliance on prior distributions.

\subsection{Instruction Following and Alignment}
Instruction-tuned models (InstructGPT~\cite{ouyang2022instruct}, GPT-4) and few-shot learners~\cite{brown2020fewshot} can follow many explicit rules, but compliance is not guaranteed when instructions conflict with pretrained priors.
Chen et al.~\cite{chen2024instructhierarchy} propose an instruction hierarchy to prioritize privileged instructions under conflict; however, their setting involves natural-language instruction conflicts rather than programming-language semantic inversions.
Xie et al.~\cite{xie2023resolving} study parametric vs.\ contextual knowledge conflicts and show that LLMs often prefer internal knowledge over instruction-given context.
Our work differs in that we study token-level and semantic-level conflicts in a programming domain.

\subsection{LLM Resistance in Educational Contexts}
The impact of LLMs on programming education has been documented in multiple empirical studies~\cite{dakhel2023github,sheard2023perceptions}.
Dakhel et al.~\cite{dakhel2023github} show that LLMs function as ``asset or liability'' depending on student expertise, and MacNeil et al.~\cite{macneil2023experiences} study LLM explanations in programming courses.
These works highlight an open problem: current assessment tasks are too easily solved by LLMs.
We address this gap by proposing a systematic language-design framework for LLM-resistant assessment.

\subsection{Adversarial Prompts and Semantic Perturbations}
PromptBench~\cite{zhu2023promptrobust} demonstrates that LLMs are fragile under adversarial text perturbations at word, sentence, and semantic levels.
Pan et al.~\cite{pan2024adversarial} show that semantically aligned adversarial perturbations in code generation prompts cause significant performance drops.
These results motivate our investigation of whether deliberately inverted semantics (a structured form of semantic perturbation) provides robust and principled LLM resistance.

\subsection{Constrained Decoding and DSL Evaluation}
Constrained decoding approaches~\cite{dong2024xgrammar,muendler2025type} enforce grammar-level validity during generation, showing that structural constraints can improve output quality.
A complementary evaluation framework~\cite{frameworkdsl2024} assessed LLM performance on constraint domain-specific languages (DSLs), finding that models perform worse on non-standard languages.
Our L2--L4 framework extends these findings by distinguishing syntactic, semantic, and implicit difficulty dimensions.

\subsection{Research Gap Addressed}
Prior work on adversarial prompts focuses on degrading general quality, not on principled language design.
Prior work on DSLs measures performance but does not explain failure mechanisms.
Prior work on instruction-prior conflict focuses on knowledge, not programming semantics.
Our contribution is a structured taxonomy (L1--L4) that characterizes LLM failure by \emph{type} (prior dominance vs.\ pattern blindness), enabling principled design of LLM-resistant educational programming tasks.
