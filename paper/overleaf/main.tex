\begin{abstract}
LLMs now solve standard programming tasks well enough to undermine their value as student assessments.
We investigate whether a human-interpretable programming language can defeat current-generation LLM next-token-prediction (NTP) priors.
We propose a four-level taxonomy---L1 (token alias), L2 (syntax inversion), L3 (explicit semantic inversion), L4 (implicit semantic inversion)---and evaluate eleven models: seven OpenAI core models (\texttt{gpt-4o}--\texttt{o4-mini}), frontier models \texttt{gpt-5.4}/\texttt{o3}, and two open-weight models (\llamaseventy{}, \qwenthirtytwo{}) via Groq.
L1--L3 provide partial or capability-dependent resistance; L4 defeats \emph{all} tested models on prior-entrenched tasks through \emph{pattern blindness}---including frontier models \texttt{gpt-5.4} and \texttt{o3} (0/10 each, PPR=1.0 on Fibonacci).
A multi-task extension (345 runs; Fibonacci, \texttt{is\_sorted}, \texttt{count\_vowels}) reveals task-dependent resistance and a third failure mode, \emph{operational substitution}.
An annotation density experiment (E34, 150 runs) yields the paper's most novel finding: \emph{pattern mimicry via code examples outperforms verbal rule annotation at every density level}, with code-example conditions achieving 50--80\% pass while fully-annotated conditions collapse to 10--30\%.
A redesigned I/O transparency experiment (E35, 60 runs) establishes I/O opacity as a second, complementary resistance pathway: opaque I/O pairs alone produce universal resistance (0/30, 0\%), independent of prior strength.
Across 1150 total L4 evaluations (11 models, 9 OpenAI + 2 open-weight), we identify three failure mechanisms: \emph{prior dominance}, \emph{pattern blindness}, and \emph{operational substitution}.\footnote{Count breakdown: 350 (strict ablation, 7 OpenAI core, $n=50$) + 75 (open-weight ablation: Llama $n_\mathrm{eff}=25$ + Qwen $n=50$) + 20 (E24 frontier: gpt-5.4 + o3, $n=10$ each) + 120 (hard-task, 4 models $\times$ 3 tasks $\times$ $n=10$) + 345 (multi-task, 7 models) + 40 (E28 annotation density gradient, \texttt{o4-mini}, $n=10{\times}4$ levels) + 150 (E34 annotation density curve, 3 models $\times$ 5 levels $\times$ $n=10$) + 50 (E32 cross-task transfer: \texttt{gpt-4o} $n=30$ + \texttt{gpt-4.1-mini} $n=10$ + \texttt{o4-mini} $n=10$). CoT ablation (160 runs) reported separately to avoid double-counting Variant~A runs. E32-ctrl control condition (10 runs, \texttt{gpt-4o}) and E33/E33v2 I/O transparency (120 runs total) excluded as confounded; E35 (60 runs) reported separately as a complementary resistance pathway.}
A chain-of-thought ablation (160 runs, 4 models) reveals a \emph{reasoning--generation dissociation}: models articulate the inversion rule when prompted to reason yet still generate standard Python code.
Open-weight evaluation confirms L4 pattern blindness with distinct failure modes and wide L1 compliance variance (KLR=0.23 vs.\ 1.00), suggesting architecture and fine-tuning govern alias resistance.
\end{abstract}
