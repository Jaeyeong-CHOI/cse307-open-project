\begin{abstract}
Recent code-oriented LLMs show strong performance on standard programming tasks, but often default to dominant language priors (e.g., Python) under ambiguous or transformed instructions. This paper studies whether LLMs follow intentionally modified Python-like syntax (``confusion language'') or revert to canonical Python grammar. We build a reproducible pipeline with strict alias validation, roundtrip transformation checks, and schema-validated metrics. Initial fixture results indicate low alias compliance and high Python reversion, motivating broader multi-model evaluation and educational implications for code verification training.
\end{abstract}
