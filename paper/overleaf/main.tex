\begin{abstract}
LLMs now solve standard programming tasks well enough to undermine their value as student assessments.
We investigate whether it is possible to design a programming language that is human-interpretable but defeats \emph{current-generation} LLM next-token-prediction (NTP) priors.
We propose a four-level taxonomy---L1 (token alias substitution), L2 (syntax inversion), L3 (semantic inversion with explicit rules), L4 (semantic inversion with implicit delivery)---and evaluate LLM compliance across thirteen models (seven OpenAI core models \texttt{gpt-4o} through \texttt{o4-mini}; frontier models \texttt{gpt-5.4} and \texttt{o3} in both hard-task and direct Fibonacci ablation; plus two open-weight models via Groq). Model coverage varies by experiment: 7 core models appear throughout all major evaluation conditions; \texttt{gpt-5.4} and \texttt{o3} appear in the hard-task battery (\S5.7) and direct Fibonacci ablation (E24); \texttt{Llama-3.3-70B} and \texttt{Qwen3-32B} appear in the open-weight evaluation (\S5.8).
L1 creates partial resistance (KLR 0.26--0.69); L2 is easily adopted; L3 reveals a model capability threshold; and L4 defeats \emph{all} tested models---including frontier models \texttt{gpt-5.4} and \texttt{o3} (0/10 each, PPR=1.0 on Fibonacci, E24)---on prior-entrenched tasks through \emph{pattern blindness}, while those same models succeed on complex algorithmic tasks (merge sort, BFS)---confirming that prior entrenchment depth, not task difficulty, governs resistance.
A multi-task extension (Fibonacci, is\_sorted, count\_vowels; 345 runs) reveals task-dependent resistance and a third failure mode, \emph{operational substitution}.
We identify three failure mechanisms---\emph{prior dominance}, \emph{pattern blindness}, and \emph{operational substitution}---across 950 total L4 evaluations spanning 13 models (11 OpenAI, 2 open-weight).\footnote{Count breakdown: 350 (strict ablation, 7 OpenAI core, $n=50$) + 75 (open-weight ablation: Llama $n_\mathrm{eff}=25$ + Qwen $n=50$) + 20 (E24 frontier: gpt-5.4 + o3, $n=10$ each) + 120 (hard-task, 4 models $\times$ 3 tasks $\times$ $n=10$) + 345 (multi-task, 7 models) + 40 (E28 annotation density gradient, \texttt{o4-mini}, $n=10{\times}4$ levels). CoT ablation (160 runs) reported separately to avoid double-counting Variant~A runs.} A chain-of-thought ablation (160 runs across 4 models) reveals that models can \emph{articulate} the inversion rule when prompted to reason but still generate standard Python code, demonstrating a reasoning--generation dissociation that is robust across dense and reasoning architectures. Preliminary open-weight evaluation (\llamaseventy{}, \qwenthirtytwo{}) shows consistent L4 pattern blindness and wide L1 compliance variance (KLR=0.23 vs.\ 1.00), suggesting architecture and fine-tuning regime govern alias resistance.
\end{abstract}
