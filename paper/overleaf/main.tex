\begin{abstract}
Recent studies on AI-assisted coding suggest a persistent gap between short-term productivity gains and durable programming comprehension. This paper proposes a reproducible benchmark for syntax-constrained code generation: language models are instructed to write Python-like programs under transformed-token rules.
\
We present an OCaml/CLI pipeline with lineage-aware task metadata, schema validation, and metric snapshot automation. Current empirical results are limited by temporary LLM API authentication failures; however, fixture replay experiments still provide a controlled baseline for measuring alias compliance, Python reversion, and execution success behavior.
\end{abstract}
