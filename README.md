# CSE307 오픈 프로젝트

이 저장소는 기존 아이디어를 고정하지 않고, **새 목표/새 방법을 탐색하기 위한 리셋 버전**입니다.

## 새로운 목표
- 특정 솔루션(언어/채점기)에 바로 고정하지 않는다.
- AI 시대에서 학습 무결성을 높일 수 있는 **실행 가능한 평가 구조**를 재설계한다.
- “완전 차단”이 아니라 “AI 단독 사용으로는 고득점이 어려운 구조”를 만든다.

## 새로운 접근 방향 (초안)
1. **평가 구조 재설계**
   - 결과물 제출 + 짧은 라이브 수정 + 구두 설명 결합
2. **개인화 과제 생성**
   - 학생별 파라미터/케이스 차등 부여
3. **과정 근거 요구**
   - 설계 선택 이유, 실패 로그, 디버깅 근거 제출
4. **운영 가능성 우선**
   - 교수/조교가 실제로 운영 가능한 난이도로 설계

## 현재 상태
- 기존 핵심 구현/설계 문서는 제거하고, 방향 재정의 상태로 전환함.
- 이후 합의된 새 목표에 맞춰 문서/코드 재구성 예정.

## 새 초안: 자연어 DSL (NL-DSL)
- 스펙 문서: `docs/nldsl-v0.1.md`
- 샘플 문항: `docs/examples/nldsl-samples.md`

이 초안은 "자연어 표면 + 형식 해석" 구조로, 다의 토큰의 문맥 기반 의미 결정을 핵심으로 한다.


## NL-DSL 파서 프로토타입
간단한 해석 로그 생성기(토이 파서)가 포함되어 있습니다.

```bash
python3 tools/nldsl_parser.py examples/nldsl-input-1.txt
python3 tools/nldsl_parser.py examples/nldsl-input-1.txt --seed-dict examples/seed-dict-sample.json
```

출력: 토큰별 의미 결정 로그(JSON)
- 적용된 섹션/모드
- 토큰 의미(sense)
- 어떤 규칙으로 결정됐는지(rule)


## 웹 인터페이스 (설명 + 제출 + 실행 결과)
`web/index.html`에서 다음을 한 번에 제공합니다.
- 파서 로직 설명
- NL-DSL 입력 제출
- seed override(JSON) 입력
- 실행 결과(JSON 해석 로그) 출력

로컬 실행:
```bash
cd web
python3 -m http.server 8080
```
