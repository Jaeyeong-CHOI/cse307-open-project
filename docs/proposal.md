# 프로젝트 재정의 메모 (Draft)

## 왜 바꾸는가
기존 초안은 아이디어 데모 성격이 강해, 실제 수업 운영/채점 관점에서 부족한 부분이 있었다.

## 새 목표
- 특정 기술 구현보다 **평가 설계 자체의 효과**를 우선 검증한다.
- 학생의 실질 이해도를 반영하는 다층 평가를 설계한다.

## 새 방법 (가안)
- 제출물 평가(정답성)
- 현장 수정 과제(이해도)
- 짧은 구두 검증(설명 가능성)

## 평가 지표
- AI/복사 제출의 통과율 감소
- 성실 제출의 오탐 최소화
- 운영 비용(채점 시간) 허용 범위 유지

## 비고
이 문서는 리셋용 초안이며, 팀/지도 의견 반영 후 정식 제안서로 갱신한다.
